# Visual Rules

## Composition

- Put the public-facing title directly at the top or visual center. Do not add production notes around it.
- Use one clear surface hierarchy. Avoid a background card containing another card containing another label.
- Give fixed-format elements stable dimensions so icons, text, and hover/focus states cannot shift the layout.
- Use full-screen compositions for video; do not shrink the actual scene into a decorative stage unless that framing has a narrative purpose.
- Prefer real plugin icons, screenshots, charts, and supplied images. Keep text and supporting evidence spatially close.

## Information Density

- A short label alone is not enough when the viewer needs to understand the tool. Include the name, one hook phrase, and a concrete capability line when space allows.
- The final overview must remain readable on the target phone or screen.
- Avoid duplicated names in both a header and an inner card unless each occurrence has a different job.
- Shorten copy before shrinking it below a practical reading size.

## Style

- Blue-white is the default when the user does not specify a palette. Pink/white, black/white, editorial paper, and dark developer styles are optional directions to propose when the content supports them.
- For editorial document videos, use full-screen white or near-white paper, restrained lines, soft cyan/pink/yellow accents, and spacious typography.
- Dark developer styling is valid only when it supports the subject; avoid generic dark sci-fi treatment.
- Do not default to dominant purple-blue gradients, decorative glow blobs, or stock-like atmosphere.
- A reference image defines direction, not a layout to reproduce.

## Aspect Ratios

- For `16:9`, keep title and core visual close to the center third; wide screens can support orbit fields, side titles, and overview cards.
- For `9:16`, make the first viewport readable on a phone; use larger title blocks and shorter card descriptions.
- For `3:4`, avoid both extremes: do not cram a full desktop board, and do not leave a tiny card floating in empty space.
- Recompose scenes for each ratio instead of scaling the same canvas.

## Motion

- Build the readable end state first.
- Use motion to explain grouping, priority, progression, or transformation.
- Reserve the strongest motion for one or two moments: orbit convergence, page turn, preview expansion, or scene transformation.
- Include quiet holds after dense entrances.
- Keep sweeps and focus glows subordinate to the content.
- Typewriter motion is optional. Offer it when the scene represents prompting, coding, searching, or text generation; do not add it everywhere.

## External-Facing Copy

Never expose internal language such as:

- “按视频冲击力排序”;
- “前 6 个 / 后 4 个”;
- “先认识这些”;
- review comments, production notes, or placeholder labels.
